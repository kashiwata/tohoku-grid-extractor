# tohoku-grid-extractor

**東北地方 ローカル系統モデル — 潮流計算・OPF 用データセット**

> Open local power grid dataset for Tohoku Electric Power Network (Japan),
> ready for power flow analysis and Optimal Power Flow (OPF).

---

## なぜこのデータセットを作ったか / Why We Built This

### 日本の系統モデルは公開されていない

日本の電力系統に関して、**基幹系統（超高圧系統）については標準モデル**が公開・共有されており、潮流計算やシミュレーション研究に広く使われている。

しかし **ローカル系統（66kV以下）のモデルは、ほぼ存在しない。**

研究目的で使える既存のローカル系統標準モデルは若干数にとどまり、特定の地域・特徴的な系統構造を対象に分析・シミュレーションをしたい場合には適していない。

### 本データセットが提供するもの

本リポジトリは、東北電力ネットワーク株式会社が公開する**実績系統図PDF**をもとに、**東北地方7地域（青森・岩手・秋田・宮城・山形・福島・新潟）のローカル系統モデル**を自動生成し、以下の形式で提供する:

- **buses.csv / lines.csv** — 系統トポロジー（バス・線路）
- **buses_pf.csv / lines_pf.csv** — 電力設定済みデータ（負荷・発電・電源種別）
- **Y-bus アドミタンス行列（CSV・npy）** — 潮流計算・OPFに直接投入可能

潮流計算（ニュートン法等）や最適潮流計算（OPF）において、モデルを一から組むことなく、**このデータセットをそのまま入力として使える状態**を目指している。

### アドミタンス行列（Y-bus）の意義

潮流計算に用いるアドミタンス行列 Y を、そのまま利用可能な形で提供する。

提供する Y-bus の仕様:
- **基準容量**: 100 MVA
- **基準電圧**: 66 kV (Z_base = 43.56 Ω)
- **モデル**: Pi モデル（対地サセプタンス含む）
- **形式**: 複素行列（実部・虚部を別CSVで提供 + numpy .npy）

## 即座に使う / Quick Start

### Python で Y-bus を読み込む

```python
import numpy as np
import pandas as pd

# 山形県の Y-bus を読み込む
ybus_real = pd.read_csv("data/05_yamagata/power_flow/ybus_real.csv", index_col=0)
ybus_imag = pd.read_csv("data/05_yamagata/power_flow/ybus_imag.csv", index_col=0)
Y = ybus_real.values + 1j * ybus_imag.values

# または numpy 形式（要 build_power_model.py の実行）
# Y = np.load("data/05_yamagata/power_flow/ybus_complex.npy")

print(f"Y-bus shape: {Y.shape}")  # (200, 200)
```

## 電力設定の仮定 / Power Settings

| ノード種別 | バス種別 | 負荷設定 | 発電設定 |
|-----------|---------|---------|---------|
| 上位系統電源（SL） | SL | — | V=1.05∠0° |
| 上位系統電源（PV） | PV | — | V=1.05, P=16MW |
| 1次変電所 | PQ | — | — |
| 配電用変電所 | PQ | P=4MW, Q=1.94MVAR | — |
| 発電所 | PQ | — | P=5MW, Q=2.42MVAR |
| 開閉所・接続点 | PQ | — | — |

- 配電用変電所の負荷: 容量 10MVA × 負荷率 40% = **4 MW**
- 1次変電所の供給: 容量 40MVA × 供給率 40% = **16 MW**（上位電源として設定）
- 発電所: 小規模再エネ想定 **5 MW**（仮置き）
- 力率: 統一 **0.90**

## 手法の概要 / How It Works

詳細は [`docs/methodology.md`](docs/methodology.md) を参照。

1. **ベクタ描画解析** — PyMuPDF でPDFの全パスを取得し、線幅・色・形状で送電線を識別
2. **シンボル検出** — 円形パスから変電所・発電機シンボルを抽出
3. **DBSCAN クラスタリング** — 端点をクラスタリングして接続点を統合 (eps=10pt)
4. **アフィン変換** — アンカー点による最小二乗フィットで緯度経度に変換
5. **Y-bus 構築** — Pi モデルによる線路パラメータ計算とアドミタンス行列生成

## 現状の課題 / Known Limitations

本ツールおよびデータセットには以下の課題がある。利用の際は留意すること。

### 系統トポロジーの精度

- **送電線の抽出漏れ**
  系統図の一部の送電線（特に密集エリア・挿入図の路線）は抽出できていない。線幅・色が微妙に異なる路線が除外されているケースがある。

- **変電所・発電所の位置精度**
  PDF上の座標を緯度経度に変換する際、アフィン変換による誤差が ±5〜20 km 程度生じる。系統図自体が模式的な表現のため、地理的に正確な位置ではない箇所がある。

- **変電所名の欠落**
  テキスト抽出（OCR）が困難な箇所や、ラベルが遠い変電所は名称が空欄になっている。

- **33kV線の過少検出**
  配電用変電所周辺の閉じたパス（変圧器バンク・母線バーなど）が33kV線と誤検出・除外されるケースがある。実際には33kV線の本数はより多いと見込まれる。

- **県境跨ぎ接続の未反映**
  地域間の連系線（県境を越える送電線）は現バージョンでは自動検出できていない。統合モデルにおける地域間接続は不完全。

### 電力パラメータの精度

- **線路パラメータが粗い**
  線路の抵抗・リアクタンス・サセプタンスは ACSR 標準値を使用しており、実際の電線種別（ACSR の規格、亘長）に基づいていない。

- **線路長の精度不足**
  PT_TO_KM（PDF座標→実距離換算）はアンカー点2点間の距離から推定しており、±30% 程度の誤差がある。系統図は距離スケールが一定でない模式表現のため、特に線路が短い・密集した地域で誤差が大きい。

- **負荷・発電量が仮置き**
  配電用変電所の負荷（4 MW）・発電所の出力（5 MW）はすべて一律の仮置き値。実際の需要や発電量を反映していない。

- **変圧器パラメータが仮置き**
  154kV/66kV 変圧器のインピーダンスは標準的な仮値（x=0.10 pu）を使用。

## 今後の改良方針 / Roadmap

### 短期（系統モデルの精度向上）

- [ ] **送電線・変電所・発電所の検出精度向上**
  未抽出の送電線・シンボルを手動補完またはルールベースで追補し、トポロジーの完全性を改善する。

- [ ] **変電所名の補完**
  系統図のラベル対応を強化し、名称の欠落を低減する。

- [ ] **線路パラメータのリアル化**
  公開されている東北地方の送電線規格データや地図上の線路長を使って R・X・B をより正確に推定する。

- [ ] **県境跨ぎ接続の自動検出・補完**
  地理座標を使った近傍マッチングの精度を上げ、地域間連系線を統合モデルに反映する。

### 中期（モデルの拡張）

- [ ] **他地域系統への展開**
  東北電力管内以外（中部・中国・九州等）の公開系統図に同様の手法を適用し、日本各地のローカル系統モデルを整備する。

- [ ] **電圧階級の拡充**
  現状の 66kV / 33kV に加え、154kV 以上の基幹系統との接続を表現する。

### 長期（時系列データの整備）

- [ ] **再エネ出力の時系列データ作成**
  太陽光・風力などの発電所について、気象データ（日射量・風速）に基づく 1 時間刻みの出力推定値を追加する。各変電所の発電量を時系列で表現することで、OPF の動的シミュレーションに対応する。

- [ ] **負荷の時系列データ作成**
  配電用変電所の負荷について、地域の需要パターン（季節・時間帯・人口分布）に基づく 1 時間刻みの推定値を整備し、潮流計算に現実的な入力を提供する。
